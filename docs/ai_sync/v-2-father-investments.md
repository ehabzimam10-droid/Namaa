# سجل تزامن الذكاء الاصطناعي (AI Sync Log) - لوحة الاستثمار العائلي للأب (v-2)

تم بنجاح بناء وتطوير لوحة إدارة "الاستثمار العائلي" (Family Investment) داخل لوحة تحكم الأب (`FatherDashboard.tsx`) مع تحديث البيانات الوهمية وتطوير نموذج تفاعلي كامل لإضافة المشاريع. يهدف هذا السجل إلى توثيق هذه التغييرات بالتفصيل باللغة العربية لمساعدة المطورين (والوكلاء الآليين) القادمين.

## 📂 الملفات التي تم تعديلها وإنشاؤها

### 1. الملفات الجديدة:
*   **[v-2-father-investments.md](file:///d:/Documents/Namaa/docs/ai_sync/v-2-father-investments.md)**:
    *   **الوصف**: هذا السجل لتوثيق التزامن وتفاصيل الكود المبني باللغة العربية.

### 2. الملفات المعدلة:
*   **[mockData.ts](file:///d:/Documents/Namaa/src/data/mockData.ts)**:
    *   **التغيير**:
        *   تعريف واجهة برمجية جديدة `FamilyProject` تحتوي على الحقول: `id`, `title`, `totalRequired`, `currentInvested`, `roiPercentage`.
        *   تحديث واجهة `FamilyData` لتشمل مصفوفة المشاريع `projects: FamilyProject[]`.
        *   تحديث كائن البيانات الوهمية `mockFamilyData` بإضافة مشروع افتراضي أول وهو: `"مشروع آلة القهوة المنزلية ☕"` بميزانية مطلوبة قدرها 1000 ريال، والمستثمر منها 200 ريال، وبعائد متوقع (ROI) بنسبة 10%.
*   **[FatherDashboard.tsx](file:///d:/Documents/Namaa/src/pages/FatherDashboard.tsx)**:
    *   **التغيير**:
        *   إضافة استيراد `FamilyProject` من ملف البيانات الوهمية.
        *   تحديث حالات المكون (State Hooks) لإدارة المشاريع الاستثمارية محلياً للسماح بالإضافة الفورية والتفاعل الكامل:
            *   `projects`: مصفوفة المشاريع المشتقة مبدئياً من `familyData.projects`.
            *   `showAddForm`: علم منطقي لعرض وإخفاء نموذج إضافة مشروع جديد.
            *   حالات الإدخال للنموذج: `newProjTitle`, `newProjRoi`, `newProjRequired`, `newProjInvested`.
        *   بناء لوحة تحكم كاملة للمشاريع الاستثمارية أسفل لوحة "المستشار المالي الذكي" بتصميم Apple Glassmorphism الأنيق:
            *   خلفية شبه شفافة مع تأثير الضباب (`bg-white/5 backdrop-blur-xl border border-white/10 shadow-2xl rounded-3xl`).
            *   زر مميز لإضافة مشروع جديد `إضافة مشروع جديد ➕` بتدرج ألوان هوية الإنماء (النحاسي والأخضر).
            *   نموذج مدخلات تفاعلي منبثق (Sleek Inline Form) يسمح بإدخال اسم المشروع والعائد المطلوب والمبالغ المخصصة مع التحقق من صحة البيانات وإضافتها فورياً للحالة المحلية وتصفير النموذج بعد الحفظ.
            *   رسم بياني شريطي مرئي (Visual Progress Bar) لكل مشروع يوضح النسبة المئوية للمبالغ المستثمرة مقارنة بالمطلوب بتدرج لوني نحاسي مائل للأخضر مع بطاقة عرض لنسبة العائد المتوقع (ROI).

## 🤖 إرشادات للعملاء القادمين (سياق العمل القادم):
*   **ربط المشاريع بالأبناء**: حالياً، تعتبر المشاريع الاستثمارية عائلية عامة على مستوى الأب. في المستقبل، قد يُطلب ربط كل مشروع بابن محدد (على سبيل المثال: أن يستثمر الطفل مدخراته في مشروع عائلي ليعود عليه بعائد). عند القيام بذلك، يمكن إضافة حقل `kidId?: string` اختياري لواجهة `FamilyProject` لتتبع مساهمة كل طفل في الاستثمار العائلي.
*   **التزامن مع الخادم**: البيانات حالياً تُحفظ محلياً في حالة React، لذا ستزول عند تحديث الصفحة. إذا تم بناء خادم مستقبلي، يجب ربط دالة `handleAddProject` بطلب POST إلى واجهة برمجة التطبيقات (API).
