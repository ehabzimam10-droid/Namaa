# سجل تزامن الذكاء الاصطناعي (AI Sync Log) - بناء الصفحات المستقلة للمشاريع والتبرعات (v-17)

يوثق هذا السجل تطبيق المرحلة الثالثة لترقية لوحة التحكم، وذلك ببناء الشاشات المستقلة والوظيفية لكل من مشاريع الأب الاستثمارية وتبرعات الأبناء الخيرية، وتوصيلها بالمحرك المالي سحابياً.

## 📂 الملفات المشمولة بالتغيير

### 1. الملفات الجديدة:
*   **[v-17-standalone-pages.md](file:///d:/Documents/Namaa/docs/ai_sync/v-17-standalone-pages.md)**:
    *   سجل التزامن باللغة العربية لتوثيق التفاصيل البرمجية والفحوصات المالية.

### 2. الملفات المعدلة:
*   **[FatherProjectsPage.tsx](file:///d:/Documents/Namaa/src/pages/FatherProjectsPage.tsx)**:
    *   استدعاء قائمة المشاريع `projects` ودالة الحفظ `addProject` من السياق العام.
    *   تصميم نموذج إدخال زجاجي متميز (Form) يحتوي على: اسم المشروع، المبلغ المطلوب، ونسبة الأرباح المتوقعة، مع زر **"إضافة المشروع ➕"**.
    *   مؤشر تحميل ذكي (800ms) وتنبيه الولي بالنجاح وتصفير المدخلات مع تحديث فوري سحابياً ومحلياً.
    *   عرض قائمة حية للمشاريع النشطة ونسبة تقدم تمويلها التراكمي.
*   **[KidDonationsPage.tsx](file:///d:/Documents/Namaa/src/pages/KidDonationsPage.tsx)**:
    *   استهلاك مصفوفة الأطفال `kids` ودالة التبرع `addDonation`.
    *   عرض شريط علوي يوضح الرصيد الفعلي المتاح للطفل ديناميكياً.
    *   بناء بطاقة مستقلة لكل قناة تبرع (سقيا ماء، كفالة يتيم، كسوة شتاء) تتيح للطفل إدخال مبلغ مخصص للعملية وزر **"تبرع للمبادرة 🤲"**.
    *   **التحقق المالي الصارم (Strict Validation)**:
        *   إذا كان المبلغ المكتوب صفرياً أو أقل، أو يتجاوز رصيد الطفل الحر، يتم تعطيل الزر فوراً وعرض النص التحذيري الموحد: **"الرصيد غير كافٍ 🚫"**.
        *   تحديث الحساب سحابياً في جدول `kids_profiles` وجدول `transactions` بـ Supabase مع بقاء مؤشر التحميل والتأكيد.
